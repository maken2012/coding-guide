⚙ Business Rules
  1. R1. When the user is not logged in → redirect to the login page and save the original request path
  2. R2. When request data exceeds pagination limit → return maximum paginated data with a warning message
  3. R3. When concurrent requests exceed the threshold → trigger rate limiting and return a 429 status code

⚠ Error Handling
Error Code Scenario Response Retry Strategy
ERR_401 Authentication failed / Token expired Return 401, guide user to re-login Not retryable, requires re-authentication
ERR_429 Request rate exceeded Return 429 with Retry-After header Exponential backoff retry
ERR_500 Server internal error Return 500, log error details Up to 3 exponential backoff retries
ERR_TIMEOUT Request timeout Return 504, indicate network error Linear retry, 2s interval
