API Contract Design

Define API specifications, request/response structures, and authentication schemes

API Overview

Method Path Description Auth
GET /api/v1/users Get user list Bearer
GET /api/v1/users/:id Get user details Bearer
POST /api/v1/users Create user Bearer
PUT /api/v1/users/:id Update user Bearer
DELETE /api/v1/users/:id Delete user Bearer
GET /api/v1/orders Get order list Bearer
POST /api/v1/orders Create order Bearer
GET /api/v1/orders/:id Get order details Bearer
Authentication Scheme: Bearer Token (JWT)
All API requests must include Authorization: Bearer <token> in the Header. Tokens are valid for 24 hours and must be refreshed via POST /api/v1/auth/refresh after expiration.
