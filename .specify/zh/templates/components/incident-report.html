Root Cause

The nightly analytics batch job was refactored in release v3.14.2 (deployed May 9) to use a shared connection pool instead of a dedicated read replica. A missing finally block in the batch processing loop caused connections to not be released after query completion. Over 8 hours, connections accumulated until the pool was fully exhausted, blocking all database-dependent services.
