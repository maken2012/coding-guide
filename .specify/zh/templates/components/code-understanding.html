💻 Client Layer

前端客户端应用，负责用户交互和界面渲染。通过 REST API 与后端通信。

  • React 单页应用，使用 TypeScript
  • 状态管理采用 Zustand
  • API 调用封装在 src/client/api/ 目录
React TypeScript Zustand
src/client/
🔌 API Gateway

API 网关层，负责请求路由、身份认证和速率限制。

  • Express.js 中间件链处理认证和日志
  • JWT Token 验证
  • 请求速率限制：每分钟 100 次/IP
Express JWT Rate Limit
src/api/gateway.ts, src/api/middleware.ts
⚙ Controller Layer

控制器层，负责 HTTP 请求解析、参数校验和响应格式化。不包含业务逻辑。

  • 输入参数校验使用 Zod Schema
  • 统一错误响应格式
  • 请求日志记录
Zod Validation
src/api/routes.ts, src/api/validators.ts
📈 Service Layer

核心业务逻辑层。所有业务规则和数据转换都在此层实现。

  • 事务管理和服务编排
  • 领域事件发布
  • 缓存策略（Redis，TTL 5min）
Business Logic Redis Events
src/core/engine.ts, src/core/processor.ts
🗃 Repository Layer

数据访问层，封装数据库操作。使用 Data Mapper 模式隔离 ORM 细节。

  • TypeORM 实体映射
  • 查询构建器封装
  • 数据库迁移管理
TypeORM Data Mapper
src/data/repositories/, src/data/entities/
💾 Database

PostgreSQL 主数据库，存储所有持久化数据。

  • PostgreSQL 15，主从复制
  • 连接池：最大 20 个连接
  • 索引优化覆盖高频查询路径
PostgreSQL Replication Connection Pool
docker-compose.yml, src/data/migrations/
