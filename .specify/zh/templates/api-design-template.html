接口契约设计

定义 API 接口规范、请求/响应结构和认证方案

API 概览

Method Path Description Auth
GET /api/v1/users 获取用户列表 Bearer
GET /api/v1/users/:id 获取用户详情 Bearer
POST /api/v1/users 创建用户 Bearer
PUT /api/v1/users/:id 更新用户 Bearer
DELETE /api/v1/users/:id 删除用户 Bearer
GET /api/v1/orders 获取订单列表 Bearer
POST /api/v1/orders 创建订单 Bearer
GET /api/v1/orders/:id 获取订单详情 Bearer
认证方案: Bearer Token (JWT)
所有 API 请求需在 Header 中携带 Authorization: Bearer <token>。 Token 有效期 24 小时，过期后需通过 POST /api/v1/auth/refresh 刷新。

GET /api/v1/users 获取用户列表
Request Headers
HeaderValueRequired
AuthorizationBearer {token}Yes
Content-Typeapplication/jsonNo
Query Parameters
ParamTypeRequiredDescription
pageintegerNo页码，默认 1
limitintegerNo每页条数，默认 20
Response

Status Codes
CodeDescription
